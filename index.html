Proyectos

A continuación, menciono algunos de los proyectos en los que he trabajado que considero más importantes o de los que más he aprendido. Regkeep - FUNIBER
Proyecto desarrollado para UNEATLANTICO, consiste en un gestor de becarios y horarios. Este proyecto lo desarrollé a petición de los directivos de manera individual durante 3 semestres, y posteriormente me convertí en gestor de proyecto con un equipo de 2 personas y en colaboración con el Centro Tecnológico Web de Santander para implementación de nuevas funciones. Este proyecto se utilizó para la gestión y toma de decisiones en diversas áreas de la universidad tales como oficinas, becas, biblioteca, residencia, entre otras areas.
Cyclope - FUNIBER
Proyecto desarrollado para FUNIBER, consiste en un micro servicio gestor de contraseñas, pensado para solventar los problemas específicos del Centro Tecnológico de Santander y posteriormente los de FUNIBER. Los roles llevados en el proyecto fueron de Programador Backend, Consultor, Arquitecto de BD y QC (Quality Control). Para este proyecto se utilizo NodeJS con Express JS y Mongo DB para el backend y React para el frontend
Canes - FUNIBER
Proyecto desarrollado para UNEATLANTICO, consiste en un micro servicio para la gestión de información de profesorado de FUNIBER y relacionados. Este micro servicio se conecta con el ecosistema de FUNIBER mediante Message Brokers y APIs. Los roles llevados en el proyecto fueron de Programador Backend, Arquitecto de Sistema y BD, QC (Quality Control) y se utilizo RabbitMQ y Lumen.
Anthos - FUNIBER
Proyecto desarrollado para UNEATLANTICO, consiste en un micro servicio conectado al ecosistema de FUNIBER que ante modificaciones de datos, realiza un registro de los datos modificados para almacenamiento histórico. Logra realizar esta tarea realizando una connexion con el ecosistema de FUNIBER mediante Message Brokers y APIs. Los roles llevados en el proyecto fueron de Programador Backend, Arquitecto de Sistema y BD, QC (Quality Control) y se utilizo RabbitMQ, Lumen y VueJs
MagicList
MagicList fue un proyecto de cuarentena desarrollado en dos meses utilizando php y Laravel. Consiste en una aplicación Web que permite a los usuarios almacenar listas en las que se podía almacenar además contenido editable con WYSIWYG, se permitía insertar recursos almacenados en S3. Y se implementó facilidades para obtener un ítem de la lista aleatoriamente, marcar ítems como completados, compartir públicamente las listas, copiar listas públicas a una cuenta, entre otras funciones. Estaba dirigido para usuarios que quisieran guardar Recetarios, guías turísticas, planes y conjunto con más información. El proyecto fue desplegado en Digitalocean con Docker y se le dio mantenimiento durante 6 meses.
DigitalCv
DigitalCV fue un proyecto de cuarentena desarrollado a lo largo de un mes utilizando php y Laravel. Consistió en un servicio web que otorgaba a los usuarios de manera gratuita un espacio o página para un curriculum Digital a partir de plantillas y completamente editable con html y la posibilidad con una suscripción, implementada con Spatie, de tener múltiples versiones digitales del currículum, cada una con un link de acceso y la posibilidad de ir intercambiando estos con el perfil principal. Se permitió a los usuarios almacenamiento de recursos en S3 para su uso, cobró digital con Spatie El proyecto fue desplegado en Digitalocean con Docker y se le dio mantenimiento durante 6 meses.
Dodge Universe
Dodge Universe es un juego para dispositivos móviles desarrollado con Unity Engine, utilizando C#. Se desarrolló durante 3 meses del 2017 en el tiempo libre en conjunto con compañero de clases encargado de el diseño de los personajes y escenarios más elaborados. El proyecto estuvo durante dos años disponible en la tienda de Google Play Store. El proyecto fue desplegado en Digitalocean con Dockery se le dio mantenimiento durante 6 meses.
Vispotify
Proyecto desarrollado en React el cual busca facilitar el visualizar los videos originales de las canciones guardadas en spotify. Conecta la cuenta de Spotify para utilizar su api de manera que detecta la canción que se está reproduciendo, posteriormente inicia un reproductor de youtube con el primer resultado con nombre de la canción y el artista obtenidos de spotify. Al acabar la reproducción se cambia a la siguiente canción mediante la api y se repite el proceso con la siguiente canción y video. El proyecto nació por el deseo de ver los videos de las canciones y se desarrolló en 3 días
